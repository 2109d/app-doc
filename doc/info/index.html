App 소개

화면(뷰)과 스크립트(콘트롤러)를 연결하여 웹 어플리케이션의 자원을 관리 할수 있도록 한다. app 인스턴스는 MakeApp 함수를 통해서 반환 받아 사용한다. 
// app 인스턴스
var app = MakeApp();
// 실행할 컨트롤러 명시
var app = MakeApp('v1');
// 다건인 경우 배열을 사용
varapp = MakeApp(['v1', 'v2']);

app.view('v1', function(ctrl){});
app.view('v2', function(ctrl){});
화면과 컨트롤러

화면과 컨트롤러를 연결하는 방법은 화면의 미리 정의된 셀렉터에 반응 하는 컨트롤러를 작성 한다. 브라우저에 이미 렌더링된 태그에 바인딩 되는 뷰와 중간에 레이어 형태로 삽입되는 팝업두가지가 존재 한다.

뷰의 경우 html 태그 엘리먼트의 data-app-view속성에 컨트롤러 이름을 작성하며, app객체의 view메소드를 이용하여 해당 뷰를 관리 하는 컨트롤러를 작성한다.

팝업의 경우 app객체의 popup메소드를 이용하여 컨트롤러를 작성하며, data-app-popup속성에 컨트롤러 이름이 추가된 형태로 엘리먼트가 생성된다. 팝업은 app실행시 생성되는 data-app-resource엘리먼트의 자식형태로 레이어가 추가된다.





  




app.view('viewName' function(ctrl) {});
app.popup('popupName' function(ctrl) {});
디자인

미리 정의된 셀렉터를 사용하여 화면을 디자인 한다.

data-app-view
브라우저에 랜더링이 완료된 화면에 디자인을 적용 할때 사용한다. 
[data-app-view] {
  width: 100%;
  background: #fff;
}
data-app-resource
브라우저의 랜더링된 영역을 간섭하지 않는 역역으로 레이어를 새로 생성하기 위한 공간이다. 스타일의 포지션 값을 설정 하여 해당 공간에 삽이되는 레이어를 관리 할 수 있다. 
[data-app-resource] {
  position: fixed;
  left: 0;
  top: 0;
}
[data-app-resource] [data-app-popup] {
  background:rgba(100, 100, 100, 0.8);
}
data-app-popup
레이어 팝업등의 새로 생성되는 화면에 사용되며, 서로 다른 스타일의 레이어 팝업을 디자인 할 때 사용한다. 
[data-app-popup] {
  position: absolute;
  width: 100vw;
  height: 100vh;
}
[data-app-popup].full >:first-child {
  width: 100vw;
  height: 100vh;
}
[data-app-popup].modal >:first-child {
  width: 90vw;
  height: 80vh;
}
[data-app-popup].modal .container {
  max-height: 80vh;
  overflow-y: auto;
}
data-app-motion
화면의 애니메이션 적용시 사용되며, data-app-motion은 뷰와 팝업에서만 동작 하는 셀렉터 이다. 시작은 open으로 종료 일때는 close값을 컨트롤러에서 세팅해주며, 이값을 이용하여 스타일시트의 애니메이션을 적용한다. 
[data-app-popup] [data-app-motion=open] {
  animation: popup-open 1s 1;
}
[data-app-popup] [data-app-motion=close] {
  animation: popup-close 1s 1;
}

@keyframes popup-open {
  0% {
    margin-top: 100px;
    opacity:0;
  }
  100% {
    opacity:1;
  }
}
@keyframes popup-close {
  0% {
    opacity:1;
  }
  100% {
    margin-top:-100px;
    opacity: 0;
  }
}
